# Next.js MVP Workflow

## Principles

- Build user flow first
- Use mock-first workflow
- Keep components small and readable
- Prefer simple state management
- Avoid premature backend complexity

## Preferred Architecture

- App Router
- TypeScript
- Tailwind CSS
- Local mock data
- Minimal dependencies

## Avoid

- over-engineering
- early database complexity
- unnecessary abstractions
- enterprise architecture patterns
