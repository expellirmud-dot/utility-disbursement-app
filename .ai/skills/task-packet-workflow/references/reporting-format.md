# Reporting Format

Every coding task must write a report under:

`.tasks/TASK-XXX/reports/`

Required report sections:

1. Task ID
2. Assigned subtask file
3. Model / tool used
4. Intended temperature
5. Files changed
6. Commands run
7. Validation result
8. Scope compliance checklist
9. Forbidden actions confirmation
10. Remaining risks
11. Commit hash
12. PR / branch reference if applicable

Reports must be factual. Do not claim validation passed unless it was actually run.
